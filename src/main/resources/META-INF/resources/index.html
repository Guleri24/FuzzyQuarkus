FuzzyQuarkus!

A fun project to learn quarkus framework and find the answer of the ultimate question: are super heroes stronger than super villains?

Why project named FuzzyQuarkus?

Just for fun.

Quarkus: Supersonic Subatomic Java

Quarkus is "A Kubernetes Native Java stack tailored for OpenJDK HotSpot & GraalVM, crafted from the best of breed Java libraries and standards". In practice, Quarkus is an Open Source stack to write Java application, specially backend application.
The memory footprint of the application build on Quarkus is very low.
Requirements:

  • Any IDE you feel comfortable
  • JDK 8+
  • GraalVM 19.2.1 or 19.3.1
  • Apache Maven (3.6+)
  • Docker
  • cURL (or any other command line HTTP client)
  • Node JS (optional, only if you are in a frontend mood)
  • most important your laptop and internet connection :)

In this project I'm going to develop an application dealing with Super Heroes (and Super Villains). For more info: Original Project by Quarkus Team.

Creating a REST/HTTP Microservice

At the heart of the Super Hero application comes Heroes. We need to expose a REST API allowing CRUD operation on Super Heroes. This microservice uses HTTP to expose a REST API and internally store data into a database. This service will be used by the fight microservice. In the following section, I learn:

  • how to create a new Quarkus application
  • how to implement REST API using JAX-RS
  • how to compose my application using CDI beans
  • how to access mu database using Hibernate with Panache
  • how to use transactions
  • how to enable OpenAPI and Swagger-UI
